Where the disagreements come from

Six distinct root causes. Counts overlap slightly where a rig fits more than one category; the table below counts each rig in its most specific category.

C1. Half-hitch on corkline rejected by canonical checkers

~20 rigs clj + rust false-red

The terminal hitch on a corkline is a half-hitch (no post twist; post: "none" in TRDL). Spec §7 (p. 19) makes half-hitches first-class. clj and rust reject them as malformed full hitches with post-key MISMATCH.

js (correct — HalfHitchInterpreter accepts) bb (correct on most) clj (false-red) rust (false-red)
spec §7 (p. 19), §7.2 (p. 21)

C2. Missing topline atom — MISSING reported as red instead of yellow

13 rigs js + bb + rust false-red

The canonical poptop / topline atom isn't bundled in the .toda file. Spec §9.1.3 (p. 30) is explicit: MISSING → yellow, not red. clj reports yellow. The other three treat the absence as a hard fail.

clj (correct — yellow) js (red — should be yellow) bb (red — should be yellow) rust (red — should be yellow)
spec §9.1.3 (p. 30), §2.5 (p. 4)

C3. bb accepts canonical-bad half-hitch rigs

13 rigs bb false-green

Across tests/toda-rig-checker/ and tests/test-suite/, bb accepts rigs designed to be bad — INVALID shields, non-fast leads/meets, untethered leads. Declared red; clj and rust correctly report red. Example: half-hitch-valid-null-shield.trdl — spec §6.1.1 (p. 17) requires lead.shld be arb-shape; setting it otherwise is INVALID → red. bb misses it.

bb (false-green) js (mixed — sometimes also false-green via HalfHitch relaxation) clj (correct) rust (correct)
spec §6.1.1 (p. 17), §9.1.3 (p. 30)

C4. Three checkers report green where canonical declares yellow

7 rigs (5 with declared-yellow agreement) js + clj + rust false-green

For 5 of the 7, bb is the only checker matching the canonical declared yellow. These are lash-and-splice edge cases (indirect-tether splicing, lashed maximal-time-crossing) where the spec answer is yellow (MISSING/UNKNOWN) per §9.1.3.

bb (correct — yellow) js (false-green) clj (false-green) rust (false-green)
spec §9.1.3 (p. 30)

C5. rust verifies known-bad rigs

2 rigs rust false-green

Two specific holes: (i) rigs/10-leadline-has-corkline-predecessor.trdl — conflicting successors; three other checkers detect this, rust says verified. (ii) tests/toda-graph/extra-fast-between-meet-and-post.trdl — declared yellow; rust says verified. Spec §8.2 (p. 27) names colinear-successors as a fundamental rigging invariant; rust is missing the check.

rust (false-green) js (correct) clj (correct) bb (correct)
spec §8.2 (p. 27)

C6. JS exceptions misclassified as FAIL/red

8 rigs (5 of which fixed; see note) js false-red on MISSING-class

The workshop's JS checker threw exceptions for atoms not bundled in the file (MissingHoistError, MissingHashPacketError) and the workshop mapped all exceptions to FAIL/red. Per spec §9.1.3 (p. 30), MISSING is yellow.

js (red — should be yellow for Missing*; fixed in app.js) clj (yellow — correct) bb (yellow — correct) rust (mixed — yellow on some, red on others)
spec §9.1.2 (p. 29), §9.1.3 (p. 30)
Note — workshop fix landed

The JS checker's run in app.js now catches errors whose class name starts with Missing and returns state: warn instead of letting them propagate to the outer pipeline's FAIL handler. Verified against the 8 affected rigs:

  • 5 rigs now correctly return yellow: cork_missing_rigging, corkline_incomplete_early_red, corkline_incomplete_early_yellow, hitch_hoist_rigs_missing, missing_rigging.
  • 3 rigs correctly stay red — they're ReqSatError (INVALID → red) and "Conflicting successors" (real spec violation): delegation-chain-4-level, invalid_rigging_green, lashed_non_colinear.

After the fix: hitch_hoist_rigs_missing and missing_rigging move to all-four-agree-yellow (clean); the others rejoin the C2 missing-topline cluster.

C7. Remaining multi-way splits

10 rigs various

Each is a unique combination of the above. Notable: complex-rig-26-lashed-complex.trdl (declared red) — only bb catches it red, the other three say green. The dominant bb-false-green pattern reverses here.
