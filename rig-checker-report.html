Where the disagreements come from

Six distinct root causes. Counts overlap slightly where a rig fits more than one category; the table below counts each rig in its most specific category.

C1. Half-hitch on corkline rejected by canonical checkers

~20 rigs clj & rust over-strict

The terminal hitch on a corkline is a half-hitch (no post twist; post: "none" in TRDL). Spec §7 (p. 19) makes half-hitches first-class. clj and rust reject them as malformed full hitches with post-key MISMATCH. js's HalfHitchInterpreter accepts them. bb mostly accepts them.

js · todajs (correct) bb (mostly correct) clj (over-strict) rust (over-strict)
spec §7 (p. 19), §7.2 (p. 21)

C2. Missing topline atom: severity mismatch

13 rigs clj is spec-correct

The canonical poptop / topline atom isn't bundled in the .toda file. Spec §9.1.3 (p. 30) is explicit: MISSING → yellow, not red. clj reports yellow, alone. js throws MissingError, bb reports red, rust reports "poptop X not found in file" red.

clj (correct) js (red — should be yellow) bb (red — should be yellow) rust (red — should be yellow)
spec §9.1.3 (p. 30), §2.5 (p. 4)

C3. bb false-greens on canonical-bad rigs

13 rigs (and ~10 mis-classified disagreements) real bb defect

Across tests/toda-rig-checker/ and tests/test-suite/, bb accepts rigs that are designed to be bad — INVALID shields, non-fast leads/meets, untethered leads, etc. The canonical declared colour is red; clj and rust correctly report red; bb reports green. One example: half-hitch-valid-null-shield.trdl — spec §6.1.1 (p. 17) requires lead.shld be arb-shape, "setting it to a shape other than arb results in an issue of INVALID"; INVALID → red. bb misses it.

bb (false-green) clj (correct) rust (correct)
spec §6.1.1 (p. 17), §9.1.3 (p. 30)

C4. bb yellow where others say green

7 rigs 5 of 7: bb matches declared

For 5 of the 7, bb is the only checker matching the canonical declared yellow — the other three over-eagerly report green. These are lash-and-splice edge cases (indirect-tether splicing, lashed maximal-time-crossing) where the spec answer is yellow per §9.1.3.

bb (correct) js / clj / rust (over-eager green)
spec §9.1.3 (p. 30)

C5. rust verifies known-bad rigs

2 rigs real rust defect

Two specific holes: (i) rigs/10-leadline-has-corkline-predecessor.trdl — conflicting successors; three other checkers detect this, rust says verified. (ii) tests/toda-graph/extra-fast-between-meet-and-post.trdl — declared yellow; rust says verified. Spec §8.2 (p. 27) names colinear-successors as a fundamental rigging invariant; rust is missing the check.

rust (false-green) js / clj / bb (correct)
spec §8.2 (p. 27)

C6. JS exceptions misclassified as FAIL/red

8 rigs workshop UX bug

The workshop maps any exception thrown by the JS interpreter to FAIL/red. Per spec §9.1.2 (p. 29), a rig checker may treat an atomic error as yellow; conditions like MissingHoistError and MissingHashPacketError correspond to MISSING issues which spec §9.1.3 (p. 30) classifies as yellow.

js (red — should be yellow for MISSING-class) workshop (label routing bug)
spec §9.1.2 (p. 29), §9.1.3 (p. 30)

C7. Remaining multi-way splits

10 rigs case-by-case

Each is a unique combination of the above. Notable: complex-rig-26-lashed-complex.trdl (declared red) — only bb catches it red, the other three say green. The dominant bb-false-green pattern reverses here.

What needs fixing

  1. bb half-hitch false-greens — 13 declared-red rigs in tests/toda-rig-checker/ and tests/test-suite/ are accepted as green. File a focused issue.
  2. Half-hitch on corkline acceptance in clj and rust — both currently reject the terminating half-hitch convention that spec §7 and §7.2 make valid. ~20 rigs affected.
  3. MISSING → yellow severity in js, bb, rust: 13 rigs where the topline isn't bundled should be yellow per §9.1.3.
  4. rust conflicting-successors check — two rigs slip through; spec §8.2 invariant.
  5. JS exception classification in the workshop — map MissingHoistError / MissingHashPacketError to WARN, not FAIL. 8 rigs.
  6. Re-probe cork_reqsat_fail.toda — the only HTTPS timeout in the run.
